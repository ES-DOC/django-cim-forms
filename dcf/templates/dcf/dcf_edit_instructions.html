Instructions for the Metadata Editing Form
  1. The purpose of the form

    This editing form is provided for users to create and modify instances of CIM Metadata Documents.

  2. The layout of the form:

    The form contains a set of tabs across the top. Each tab corresponds to a particular category of properties. The numbers on the tab correspond to the total number of properties left to update within that category.

    Each property may contain an optional help icon with useful information. Properties that are required are indicated by a bold font. There are several different types of properties:
    • "atomic" properties: simple things like strings, integers, dates, etc.
    • enumerations: properties whose values are restricted to a list of options. Enumerations may be "closed" (ie: completely restricted to the set of options presented) or "open" (ie: the user can select the option "OTHER" and then type in their own value). They can be restricted to a single value or multiple values. And they can include an explicit "NONE" value.
    • relationships: properties that define a mapping from the current CIM type to another CIM type.
    • scientific properties

  3. A form can be saved and published. Saving simply stores the current state of the form to the local database. Publishing serializes the form to a CIM Document and exposes it in the built-in ATOM feed. Other tools and services can then ingest the document (the CIM Viewer, for instance).
