Instructions for the Metadata Customization Form
  1. The purpose of the form

    This customization form is provided for project administrators to customize the look-and-feel of the metadata entry forms used by their project members. The CIM is a highly complex model with a vast hierarchy of potential descriptive metadata for any given artifact. To require all users to input all CIM metadata is impractical. Similarly, the CIM is a very generic metadata standard and the metadata element names and other related information may not be suitable for all users. This form, therefore, provides a means to tailor metadata presentation to a given audience.

  2. The layout of the form

    The form contains three sections:

    1. a collection of fields that describe the current customization:
    2. a widget containing all potential field categories in the model:
    3. a fieldset containing all potential fields in the model:

    The Categories Widget is used to categorize each field. These categories correspond to the tabs that appear across the top of the actual metadata entry form. This widget functions very similarly to a standard "tagging" widget. To add a new category simply type in the new category name into the widget. To delete a category click on that categories "x" icon. To edit a category (ie: to add a text description of the category), click on the "pencil" icon. Note that deleting a category is permanent. This means that any fields which have been associated with that category will no longer be associated with any category.

    To reorder the categories, which corresponds to reordering the tabs of the metadata entry form, simply drag and drop the categories within the widget to the desired order. This customization form defines two orders:

    1. the order of field categories
    2. the order of the fields themselves (editing this is described below)

    Because this can be confusing, administrators can restrict the fields that are displayed to a specific set of field categories. To toggle field displaying on and off, click on the "lightbult" icon in the field categories. When this is active, all fields of that category will be displayed. When it is inactive, no fields of that category will be displayed.

    The Fields Widget is used to customize each field. All fields in the CIM Model are displayed in an accordion. The accordion header gives basic field information:

    1. the field name (as defined by the CIM)
    2. the field type (as defined by the DCF database)
    3. the field category (as defined by the administrator, using this form)

    Clicking on the header opens or closes the advanced field view for that particular field. Additionally, there are global "expand all / collapse all" buttons which can be used to open or close all displayed fields.
