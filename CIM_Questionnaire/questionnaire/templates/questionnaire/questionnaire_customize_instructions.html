The CIM Questionnaire customization process
  1. The purpose of the form

    The CIM Questionnaire customization form is provided for project administrators to customize the look-and-feel of the CIM Questionnaire editing forms used by their project members. The CIM is a highly complex metadata standard with a vast hierarchy of potential content for any given artifact. To require all users to input all CIM metadata is impractical. Similarly, the CIM is a very generic metadata standard and the metadata element names and other related information may not be suitable for all users. The customization form, therefore, provides a means to tailor metadata presentation to a given audience.

  2. The use of the form

    The customization form creates customizers for particular combinations of metadata versions, model types, and specific projects. To access a particular customizer the following syntax is expected:

    http://your.domain.com/<project>/customize/<version>/<model_type>
    http://your.domain.com/<project>/customize/<version>/<model_type>/<existing_customization_name>
    However, it is recommended to use the project's "index page" to choose from the available customizers:
    http://your.domain.com/<project>
    Although several customizers may exist for a given project/model/version combination, only one of those - the default one - will be used by the CIM Questionnaire editing form.
