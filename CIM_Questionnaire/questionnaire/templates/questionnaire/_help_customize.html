What is Customization?

The CIM is a large and complex metadata standard. Exposing its full complexity in the Editor would be overwhelming to end-users. For this reason, the CIM Questionnaire allows project administrators to customize the look-and-feel of the Editor for each different document type that might be created. This includes restricting the set of both standard and scientific properties that are displayed, changing what is required and optional, changing how properties are labeled and what their documentation is, and so on.

Logging In

You must be logged into the CIM Questionnaire to create or edit a Customization. You can request a CIM Questionnaire account here. After receiving an account, you must still join the particular project you wish to create customizations for as an administrator.

Creating a New Customization

To create a new customization, first navigate to the project that you are interested in. From that page, select the appropriate ontology and document type under "New Customizations" and then click the "create" button. This will bring you to the CIM Questionnaire Customizer.

Editing an Existing Customization

Again, to edit an existing customization, first navigate to the project that you are interested in. The "Existing Customizations" section lists the names of customizations that have already been created for that project.

These are organized by ontology version and document type. Additionally, an asterisk appears after the name of the default customization.

You may edit the customziation by clicking "Edit" next to that customization. Please do not have more than one editing session open for any given customization.
