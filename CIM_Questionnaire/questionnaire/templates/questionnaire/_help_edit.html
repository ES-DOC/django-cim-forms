Logging In

You must be logged into the CIM Questionnaire to create, edit, or publish CIM Documents. You can request a CIM Questionnaire account here.

After receiving an account, you must still join the particular project you wish to create documents for. Without being a member of the project you will only be able to "view" and not "edit" documents.

Creating a New Document

To create a new document, first navigate to the project that you are interested in. From that page, select the appropriate ontology and document type under "New Documents" and then click the "create" button. This will bring you to the CIM Questionnaire Editor:

Editing an Existing Document

Again, to edit an existing document, first navigate to the project that you are interested in. The "Existing Documents" section lists the names of CIM Documents that have already been created for that project. You may edit the document by clicking "Edit" next to that document. (If you click "View in Editor" you will only be able to view the document, and will not be able to save any changes.) Please do not have more than one editing session open for any given document.

The Editor Form used for creating and editing a CIM Document is exactly the same.

The Editor Layout

There are several distinct sections to the CIM Questionnaire Editor.

Component Hierarhcy

This is a clickable tree where each branch corresponds to a sub-component of the model being documented

Component Panes

To the right of the Component Hierachy are the Component Panes. These contains all of the properties belonging to the currently selected component. There are two different categories of properties to deal with: standard and scientific.

Standard Properties

The first three tabs along the top of the Component Pane contain the standard properties of the component. These are properties that come from the CIM Schema. They are the same for every single component.

Because many standard properties would have the same value for each component, a standard property can be inherited. This icon indicates an inherited property: As the value of a component's inherited property changes, that value propagates to the corresponding properties of all child components. To disable this feature, simply uncheck the inheritance icon.

Scientific Properties

The final tab, the one that says "Scientific Properties" contains, unsurprisingly, the scientific propoperties of the component. These are the properties that come from the CIM Controlled Vocabulary. Whereas standard properties are the same for each component, scientific properties vary. The second level of tabs within the Scientific Properties Pane (the one that begins "Mediation Services" in the above screenshot), shows the distinct categories of scientific property for the selected component.

Entering Information

Simply fill in a value for every property of each component.

You should start at the highest level of the Component Hierarchy.

Fill out all of the standard properties ("Document Properties" and "Basic Properties" and "Component Description"). Note that, in most cases, the "Short Name" property in the "Basic Properties" tab becomes the name of your new document once you save it. And be sure to take advantage of inheritance, as described above.

Next, for each component, fill out all of the scientific properties ("Scientific Properties"). Initially, the input allowing you to change a scientific property's value is hidden within a collapsed accordion. It is easiest to use the "expand all" button to be able to edit all of the properties that still need values.

Saving Your Work

You may save your partially complete document by clicking the "save" button at the bottom of the page while you are editing. It is not necessary to do this after every change, but it is recommended to save frequently.

Every time you click "save", a new minor version of the document is saved to local storage (Example: version 0.31 -> 0.32).

Publishing a Document

Please do not publish your document until it has reached a level of completion that you are comfortable with sharing.

Publishing puts the data into a publicly accessible data feed. Clicking "publish" for a document that has already been published creates a new major version, that is, version 1.x -> 2.1. (An unpublished document is denoted version 0.x.)

A CIM document must be published in order for the ES-DOC Viewer and Comparator to access them.
